ticket-refinery — onboarding

Audience: new teammate. Goal: run, debug, and safely change the pipeline before your first PR.

1. What this is

Containerised Python pipeline that polls Azure DevOps for work items tagged needs-refinement, shallow-clones the repos named on the item into a per-item scratch workspace, asks the Pi coding agent to refine the item into structured findings, validates those findings, then writes them back to ADO (description patch, AC patch, summary comment, attachment, tag transition). One CLI command, runs in Docker, exit code is the success signal — not the stdout.

Mental model: the orchestrator does everything boring (clone, build curated repository context, render prompt, call Pi, validate, publish, clean up). Pi only reasons over the provided context — it has no exploration tools and re-discovering the repo itself is forbidden by the prompt.

4. Repo map

Orchestration

  • src/refine.py — thin orchestrator: env load, DI, queue loop, back-compat process_item wrapper.
  • src/services/refinement_service.py — the per-item pipeline (the only seam that touches metrics). 7 phases; cleanup is unconditional in finally.

Services

  • src/services/workspace_service.py — clone, cache symlink, Graphify re-index, cleanup. Broken-symlink handled (unlink + recreate).
  • src/services/context_service.py — load existing comments + render the prompt (accepts an optional repo_context_section).
  • src/services/publishing_service.py — all ADO writes (description/AC/title inside a marker block, comment, attachment upload, tag transitions). Has an open AC stub — see dragons.

Leaf modules

  • src/ado_client.py — ADO REST (WIQL, JSON Patch, comments, attachments). Dumps non-JSON responses to /tmp/ado-non-json-response.html on proxy/auth failure.
  • src/git_ops.py — parallel shallow clone. PAT delivered via short-lived GIT_CONFIG_COUNT + http.extraHeader per clone (never in URL).
  • src/pi_runner.py — Pi subprocess wrapper with retry on transient errors. Raises InfraError on non-JSON or non-zero exit (no retry, parser won't suddenly succeed).
  • src/validate.py — JSON Schema (Draft-07) + per-sourceRef file existence check. Never retried — schema/ref failures are permanent.
  • src/retry.py — centralised 3-attempt backoff (1s/2s/4s). Opt-in by exception class tuple.
  • src/metrics.py — counters + timer context manager + immutable MetricsSnapshot.

Repository intelligence

  • src/repository_knowledge.py — RepositoryKnowledge facade over KnowledgeBackend. Implementations: GraphifyBackend (default, reads parsed AST graph directly) / FilesystemBackend (grep fallback, all curated ops return degraded=True).
  • src/repository_context.py — RepositoryContext DTO + RepositoryContextBuilder. Entity extraction is code-shaped (must have internal capital / underscore / digit) and explicitly filters EN + DE stopwords. Called by the orchestrator before Pi runs.
  • src/prompts/refine.prompt.tmpl.md — the prompt. {repo_context} is replaced by the orchestrator with the curated block; {schema} is the live JSON schema appended verbatim.
  • src/schema/findings.schema.json — output contract.
  • src/repos.jsonc — repo registry (repo:<name> → url + defaultBranch). Currently only one entry. Mounted :ro from host.

5. The 5 things to understand cold

  1. src/refine.py:main() — thin. Loads Config from env, builds services once, calls client.query_items(...), iterates items. Exit-code semantics are the whole game: pi_runner.InfraError → exit 1; validate.ValidationError → blocked tag + exit stays 0 (pipeline worked, content didn't); any other Exception is uncaught and exits 1 with a traceback.
  2. RefinementService.refine() — the 7-phase pipeline. Cleanup always runs in finally; metrics live here (counters for success/blocked/infra-failure, timers per phase). The _record_duration helper is rough — see dragons.
  3. PublishingService — every ADO write. _publish_done patches Description + AC (+ optional Title) then comments + uploads attachment + transitions tag. _publish_blocked posts unknowns-list comment and tags blocked. Attachment upload runs first so a successful publish never half-completes.
  4. RepositoryKnowledge — the facade + two backends. Curated ops on the base class return degraded=True defaults; GraphifyBackend is what makes them real by reading graph.json directly (no per-query CLI). _load_graph handles both single-repo and workspace-of-cloned-repos layouts.
  5. RepositoryContextBuilder — the curated kickstart. Defensive: every backend call is wrapped in _safe_* so a broken Graphify never breaks the prompt. Renders a small markdown block (architecture, /graphify cheatsheet, relevant files) — full dep graph stays on disk.

6. Setup & first run

Setup (one-time)

  1. Create .env from .env.example and fill in ADO settings + tags + PI_MODEL.
  2. Ensure ~/.pi/agent/auth.json exists (subscription OAuth tokens).
  3. Add the repos you'll refine to src/repos.jsonc (repo:<name> → url + defaultBranch).
  4. Docker or Podman on PATH (Podman rootless needs the DNS fix in run.ps1).

Graphify and Pi ship inside the container image — the host doesn't need them.

First run

./check.ps1          # preflight: .env present, no inline #, auth.json, image, container engine
./run.ps1            # build local image, run, propagate exit code
./run.ps1 -UseRemoteImage   # pull ghcr.io/moeller-projects/ticket-refinery:latest
./run.ps1 -Build            # force rebuild local image

8. Validation & exit codes

ExitMeaning
0queue drained or every item reached refinement-done / refinement-blocked
1infra failure: auth, clone, Pi invocation, unhandled exception
2launcher failure: missing .env, no container engine

Reaching refinement-blocked (unknowns, validation failure) is a successful run. Only auth / clone / Pi failures fail the container.

10. Conventions / fingerprints

  • ponytail: comments mark deliberate simplifications and "this exists because X" notes. Read them before reading the function — they are faster and more honest than the code they annotate.
  • Module-attribute imports at boundaries that tests monkeypatch: import git_ops, import pi_runner, import validate (never from X import Y at those seams).
  • if __name__ == "__main__" self-checks at the bottom of leaf modules: framework-free, fail-fast, one-shot.
  • from __future__ import annotations + dataclass(frozen=True) everywhere. No mutable shared state.
  • Retry is opt-in by exception class, not opt-out. Every retrying site passes its own tuple.

11. Gotchas & dragons

These will bite you if you don't know they're here.

Operator-level gotchas

  • Inline # comments in .env break parsing — check.ps1 flags them. (This was a real failure mode.)
  • Podman rootless often fails DNS resolution inside the container — run.ps1 forces --dns 8.8.8.8 + 1.1.1.1 on podman.
  • PAT scope must be Work Items Read & Write + Code Read. Wrong scope → HTTP 401 with an HTML body, dumped to /tmp/ado-non-json-response.html for debug.
  • PAT must never end up in a clone URL — git_ops.py uses short-lived GIT_CONFIG_* auth headers per clone.
  • Pi subscription OAuth tokens at ~/.pi/agent/auth.json are mounted read-write in run.ps1 so token refresh works on long runs. Mounting :ro breaks renewal.
  • Missing repo:<name> in src/repos.jsonc is a hard fail (raises InfraError), not a guess or skip.
  • Unresolved sourceRef paths never retry — schema / ref failures are permanent; only transient HTTP / subprocess failures are retried.
  • ALLOW_TITLE_EDITS=true lets Pi rewrite work-item titles — off by default, dangerous.

Code-level dragons

  • findings_to_ac_html is a stub in publishing_service.py that writes a literal placeholder into the AC field. Until the AC shape is decided, every successful publish gets a placeholder AC.
  • _record_duration in refinement_service.py defines an inner _on and immediately abandons it, returning _record which writes to the private MetricsCollector._timings_ms directly. Comment: "private but only this class writes it." Works today, will silently break the moment any second metrics-writing class lands.
  • One uncaught-exception gap: RefinementService.refine() only knows how to handle pi_runner.InfraError and re-raises everything else. main() catches InfraError and ValidationError explicitly. Anything else (RuntimeError, OSError, etc.) bubbles up uncaught → exit 1 with traceback.
  • publishing_service._safe catches Exception broadly inside per-write retry. If the comment fails after retries, the function logs and continues, but add_tag(blocked) still fires afterwards — work item ends up tagged blocked with no human explanation.
  • Working tree is dirty: src/services/refinement_service.py, LICENSE, .gitignore all show CRLF-only diffs against HEAD. Looks like an editor or .gitattributes change. Confirm before any further work.
  • repos.jsonc has one entry (Laekkerai.Ordering). Every new repo requires an operator edit + container restart. File is mounted :ro from the host.
  • _extract_entities is harsh to non-code text. Tickets titled entirely in German/English prose produce an empty entity list → relevant_files=None in the prompt. Documented behavior, worth knowing.

12. Open questions (worth asking a teammate)

  • The uncommitted CRLF diff — accidental editor swap, or about to commit a .gitattributes pinning LF?
  • Is the findings_to_ac_html stub intentional (waiting on AC shape spec) or forgotten?
  • Auto-retry on refinement-blocked is on the README's open-questions list. Decision yet, or still manual retag?
  • Why is repos.jsonc mounted :ro rather than baked into the image?
  • The Bearer-token priority chain (SYSTEM_ACCESSTOKEN → ADO_AUTH_TOKEN → ADO_MCP_AUTH_TOKEN → ADO_API_KEY) is documented in .env.example but AdoClient.__init__ only takes one PAT. Is there an alternate auth path coming, or dead documentation?
  • The pre-existing test_ado_client.py failure — anyone planning to fix it, or frozen as "known broken, unrelated"?
  • Single ADO project assumed — does the org use multiple ADO projects with overlapping repos?
  • What does the production Pi subscription model actually do end-to-end (subscription auth, model fallback, tool profile)? The repo can't answer this; needs a real run.
