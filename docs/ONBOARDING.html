ticket-refinery — onboarding handoff

Audience: new teammate. Goal: run, debug, and change pipeline safely.

What this is

Containerized pipeline that reads ADO work items tagged needs-refinement, clones repos, asks Pi to produce structured findings, validates output, writes back description / acceptance criteria, comments, and moves tags.

System overview

flowchart LR A[.env + auth.json] --> B[run.ps1] B --> C[container] C --> D[src/refine.py] D --> E[ADO REST] D --> F[git clone] D --> G[Pi CLI] D --> H[src/validate.py]

Repo map

  • src/refine.py orchestrator
  • src/ado_client.py ADO REST
  • src/git_ops.py shallow clone helpers
  • src/pi_runner.py Pi subprocess wrapper
  • src/validate.py JSON schema + sourceRef checks
  • src/prompts/refine.prompt.tmpl.md Pi prompt
  • src/schema/findings.schema.json output contract
  • src/repos.jsonc repo registry

Setup

  • Create .env from .env.example
  • Set ADO_ORG, ADO_PROJECT, auth, tags, PI_MODEL
  • Ensure ~/.pi/agent/auth.json exists
  • Use Docker or Podman

First run

./check.ps1
./run.ps1

Pipeline flow

sequenceDiagram participant H as Host participant C as Container participant A as ADO participant P as Pi H->>C: run with .env + auth.json C->>A: query tagged items C->>C: clone repos + build prompt C->>P: pi -p --tools read,bash,grep,find,ls P-->>C: JSON findings C->>C: validate schema + sourceRefs C->>A: patch description / AC + comment

ADO contracts

  • Input tag: needs-refinement
  • Success tag: refinement-done
  • Blocked tag: refinement-blocked
  • Permissions: Work Items Read & Write + Code Read
  • Repo mapping: repo:<name> tags must exist in src/repos.jsonc

Pi integration

Pi runs as CLI subprocess with read-only tools: read,bash,grep,find,ls. It must emit JSON that matches src/schema/findings.schema.json.

Config reference

Main vars: ADO_ORG, ADO_PROJECT, ADO_PAT, TAG_TRIGGER, TAG_DONE, TAG_BLOCKED, CLONE_DEPTH, PI_MODEL, PI_TIMEOUT_SECONDS, TARGET_LANGUAGE, CONTAINER_ENGINE.

Validation & exit

  • 0 queue drained / handled
  • 1 infra failure
  • 2 launcher failure

Gotchas

  • Inline # comments in .env break parsing.
  • Podman rootless may need forced DNS.
  • Missing ADO tag or custom field breaks the run.
